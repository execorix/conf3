Инструкции сгруппированы по функциональности. Поля команд (B, C) определяются в зависимости от типа операции.

A. Арифметико-логические Операции (ALU)
ADD (0x3), SUB (0x4): Двухадресные операции (Register-Register). Результат записывается в target_reg.

Пример: {"op": "ADD", "target_reg": "R1", "source_reg": "R2"} (R1 = R1 + R2)

B. Передача Данных (Data Movement)
LDI (0x9): Загрузка константы (value) в регистр (target_reg). Использует 26-битное поле константы.

Пример: {"op": "LDI", "target_reg": "R5", "value": 1000}

LOAD (0xC): Чтение из памяти по адресу (addr) в регистр (target_reg).

Пример: {"op": "LOAD", "target_reg": "R3", "addr": 500}

STORE (0xA): Запись из регистра (source_reg) в память по адресу (addr).

Пример: {"op": "STORE", "addr": 600, "source_reg": "R4"}

NEQ (0x2): Логическая операция "Не равно". Сравнивает регистр (target_reg) со значением в памяти (addr). Результат (1, если не равны; 0, если равны) записывается обратно в регистр.

Пример: {"op": "NEQ", "target_reg": "R0", "addr": 10}

C. Управление Потоком (Control Flow)
JMP (0x5): Безусловный переход к адресу (addr).

Пример: {"op": "JMP", "addr": 200}

JZ (0x6): Условный переход к адресу (addr), если регистр-условие (condition_reg) равен нулю.

Пример: {"op": "JZ", "condition_reg": "R8", "addr": 300}

D. Ввод-Вывод (I/O)
IN (0x7), OUT (0x8): Используют target_reg для данных и value_code для указания канала/кода I/O.

Пример: {"op": "OUT", "target_reg": "R5", "value_code": 1}

Пример: {"op": "IN", "target_reg": "R6", "value_code": 10}

E. Прочие
NOP (0x0): Отсутствие операции.

Пример: {"op": "NOP"}
